# Studio-Variante — Implementierungsplan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Dritte, parallele Dashboard-Oberfläche „Studio" unter `/studio` — hell/dunkel umschaltbar, Desktop-Seitenleiste, mobile Bottom-Navigation, Startseite mit Signalen, KPIs, Verlauf, Sparten-Vergleich und Kategorien-Ranking.

**Architecture:** Reines Frontend in neuem Verzeichnis `static-studio/` (index.html, style.css, charts.js, app.js, fonts/), als dritter StaticFiles-Mount in `app/main.py`. Keine neuen Endpoints; Signale/Vorjahresvergleich werden im Browser aus `/api/verlauf`, `/api/jahresvergleich` (per_kategorie) und `/api/dashboard` berechnet. SVG-Graphen selbstgebaut nach dem Muster von `static-cockpit/charts.js`.

**Tech Stack:** FastAPI-Static-Mount, Vanilla JS (ES2020), CSS Custom Properties, Inline-SVG, lokale WOFF2-Fonts. Keine Fremd-Libraries.

## Global Constraints

- Spezifikation: `docs/superpowers/specs/2026-07-14-studio-design.md` (freigegeben 2026-07-14)
- `/` und `/cockpit` bleiben unangetastet; keine Schema-/DB-Änderungen
- Tests NUR gegen Wegwerf-DB via `FINANZ_DB`-Umgebungsvariable (echte NAS-DB nie anfassen)
- Keine externen Requests (Fonts lokal, kein CDN)
- Sparkline/Graph darf nie Zahlen überdecken (Cockpit-Feedback)
- Theme: Standard `prefers-color-scheme`, Umschalter, Persistenz in `localStorage` (`studio-theme`)
- Mobile Breakpoint: 900 px; Touch-Ziele ≥ 44 px
- Assets mit `?v=1`-Cache-Buster einbinden
- Sprache der UI: Deutsch; Beträge als `1.234,56 €` (Cent-Integer aus API, Formatierung wie `centToEuro` im Cockpit)

### Task 3: Graph-Bibliothek (charts.js)

**Files:**
- Create: `static-studio/charts.js` (Muster: `static-cockpit/charts.js`)

**Interfaces:**
- Produces: globales `Charts`-Objekt mit
